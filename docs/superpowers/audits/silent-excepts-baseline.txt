Why this file exists
--------------------
Trip 41 + trip 43 both burned hours debugging production issues that
turned out to be silent `except Exception: pass` blocks swallowing the
real error. Tier 2 of the bug-proofing plan adds an AST linter that
catches the pattern. Tier 3 will gate CI on "no NEW silent excepts" —
this file is the baseline we diff against.

We are NOT fixing all 19 in this commit. They were intentional or
not-yet-prioritized; cleaning each one needs a per-call-site judgment
(swallow vs log vs re-raise vs surface to caller). What we ARE doing
is freezing the count: any future commit that adds a silent except
must justify itself or be rejected.

If the count goes UP, the offending commit must either:
  1) log the exception with context, or
  2) re-raise (optionally as a domain error using `from e`), or
  3) catch a narrower exception (e.g. `except KeyError:`), or
  4) document why this site is genuinely fire-and-forget and update
     this baseline in the same commit.
